EmberJS

Intro

What's EmberJS?

  • A JS MVC framework
  • Comes with a CLI called (surprise) ember-cli
  • Templates written in Handlebars
  • CSS by default but Sass / Compass / Less / Stylus installing npm modules
  • Has ES2015 built in, but you can use CoffeScript via npm module

Basic installation

What's needed?

  • Node 4.x.x
  • Bower

Ember-cli

> npm install -g ember-cli

Extra's installation

Sass

> ember install ember-cli-sass

PhantomJS

> npm install -g phantomjs

Project initialization

Init

This will create all the basic scaffolding and install all the required packages.

> ember new app-name

Dev server

> ember serve

Ember schema

Config

Inside config/environment.js is our app configuration.

To avoid XSS scripting, diferent basepaths than the root are forbidden. To include external sources like fonts, styles or libraries, you should specify them on ENV.contentSecurityPolicy property.

contentSecurityPolicy: {

'default-src': "'none'",

'script-src': "'self' 'unsafe-inline' 'unsafe-eval'",

'font-src': "'self' https://fonts.gstatic.com",

'img-src': "'self'",

'report-uri':"'localhost'",

'style-src': "'self' 'unsafe-inline' https://fonts.googleapis.com",

'frame-src': "'none'"

}

Ember schema

Common elements

  • Route: creates an accessible path on router.js a js file inside routes/ and a view file inside templates/
  • Controller: needed as a concept to work (by now), but not needed to exist as a file on controllers/ so only generate one if the route's view needs own logic.
  • Model: Data model for a route.
  • Component: ember's bet to substitute routes and controllers. It will be created a file in components/my-component.js and templates/components/my-component.hbs as its view.
  • Helper: template helper to process data on the view side.
  • Mixin: a bunch of methods to inject in controllers or components

Every created element becames with the default test blueprint file.

Routes

Examples

Router.map(function() {

this.route('elements', { path: '/elements' });

this.route('elements');

this.route('element', { path: '/element/:element_id' });

this.route('groups', function(){

this.route('index');

this.route('special');

});

});

  • There is a shorthand method to define routes on route.js
  • Parameters on routes are defined without the shorthand.
  • By default, every route has an internal index.hbs view to render on '/' (including the main App route)
  • A controller is created for every route, if there is no real controller created, ember would use the simplest one.

Controllers

  • Just used in case the app needs logic on a route.
  • They are going to be removed on the near future by using routable components.
  • As long as the data model can only be picked on routes, it will be present.

Components - setUp

components/poc-component.js

export default Ember.Component.extend({

tagName: 'ul',

classNames: ['component col-xs-6-12'],

classNameBindings: ['isActive:selected'],

value: 2,

isActive: Ember.computed(function(){

return this.selectedValue === this.value

}).property('selectedValue','value')

tagName: 'ul',

});

  • Components have to have a dashed name.
  • By default, components are rendered in a div tag.
  • classNameBindings have a shortcut mode with isNAME.
  • Attributes passed in template call, don't need to be declared inside.

Components - actions

components/poc-component.js

export default Ember.Component.extend({

actions: {

changeSelectedValue(index){

this.set('selectedValue',index);

this.sendAction('magic'); (keep in mind this line)

}

},

click(){

alert('Do I annoy you?');

}

});

  • Actions are present in controllers too.
  • Declare any data/user interaction here.
  • Can be aquired by its parent, and triggered locally.
  • There is a built in events HERE

templates/components/poc-component.hbs

{{#each list as |list index|}}

<li {{action "changeSelectedValue" index}}> {{index}} {{list}} </li>>

{{/each}}

<li>{{yield}} </li>

  • Components have to have a dashed name.
  • By default, components are rendered in a div tag.
  • classNameBindings have a shortcut mode with isNAME.
  • Attributes passed in template instance don't need to be declared inside.
  • Components can be caled as block elements. Content placed inside will be displayed on {{yield}}

Mixins

  • Group of functions to inject into components/controllers

mixins/time-relative.js

export default Ember.Mixin.create({

hourBuilder : function(timestamp){

let time = new Date(timestamp);

let minutes = time.getMinutes().toString();

minutes = (minutes.length<2) ? '0'+minutes : minutes;

return time.getHours()+':'+minutes;

}

});

components/poc-component.js

import TimeRelativeMixin from '../mixins/time-relative';

export default Ember.Component.extend(TimeRelativeMixin,{

timestamp : new Date(),

hour : Ember.computed(function{

return this.hourBuilder(this.get('timestamp'));

}).property('timestamp')

});

Downsides solved. Helpers

  • Templates are constrained by Handlebars restrictions regarding on moving away logic from views.
  • Being a No-Logic-in-Views fanatic make simpler things a headache.
  • Helpers can be chained one inside another.

Basic helpers I needed

  • Inline if: If is a block element, not alowing you to use it in attributes.
  • And / Or logic operators: If only accepts boolean values, pretending to create a third computed variable between the compared pair. Creating repeated code each time needed.
  • Eq: As If only accepts boolean values, eq accepts a pair and compare them.
  • Math: Simple math is not allowed by default, even to concatenate strings.
